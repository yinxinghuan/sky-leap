# Requirements

## 1. Overview

Sky Leap 是一款单手蓄力跳跃游戏：玩家带着自己装备的像素通勤者，在悬空站台间跳跃、收集车票，并解锁六位可收藏角色，而不是每局被随机分配角色。

## 2. Visual Design

- 主画面为竖屏 WebGL 云海与悬空站台；角色位于画面中下部，下一站台始终在前方可读区域。
- 收藏入口固定在左上安全区，包含当前装备角色的 SVG 剪影和车票总数；排行榜保留在右上安全区。
- 角色商店采用“站台衣柜”底部抽屉：中央大尺寸真实 3D 角色预览，左右同时露出相邻角色；名称、已装备/未解锁/可解锁状态和购买动作围绕当前角色组织，避免随机抽卡或盲盒表达。
- 角色均使用与游戏模型相同的低多边形、方块剪影语言。功能图标使用内联 SVG，不使用 Emoji。
- 窄屏宽度 320 px、高度 568 px 时完整显示当前角色、前后切换、状态、解锁按钮和关闭按钮；所有可点击控制至少 44 px。

## 3. Game Mechanics

- 蓄力上限为 1.05 秒；跳跃距离为 2.5–11.5 单位，高度为 1.2–3.3 单位，空中时间为 0.42–0.72 秒。
- 成功落点得 1 分；落在目标中央 30% 区域时得 3 分并增加连击；中间 40% 区域得 1 分并保留连击；边缘落点得 1 分并重置连击。
- 每局结束按 `min(8, floor(score / 4) + 1)` 获得 1–8 张车票，分数为 0 也获得 1 张，保障新玩家持续解锁。
- 初始拥有“早班店长”。角色商店的解锁顺序和成本为：赶课学生 10、夜班主厨 25、闪送骑手 45、消防英雄 70、末班幽灵 100 张车票。
- 已解锁角色可免费反复装备；装备会保存到本地，重开或刷新后保持。每次重开使用当前装备角色，禁止随机更换。
- 碰撞以角色脚印和站台中心距离判定：落点距站台中心不超过 `站台半长 + 0.32` 即成功，否则坠落。
- 命中、完美命中、坠落分别产生 4–26 个粒子；单次特效持续 0.30–0.90 秒。

## 4. Controls

- 游戏画面：Pointer down 开始蓄力，Pointer up / cancel 立即起跳；商店、排行榜和结算层不会触发蓄力。
- 收藏入口、角色卡和结算页的收藏按钮：Click 打开或操作角色商店。
- 未解锁角色卡：车票足够时单击同时解锁并装备；不足时保持锁定，不扣除车票。
- 已解锁角色卡：单击立即装备；只在待机或结算状态替换画面角色。

## 5. Win / Lose Conditions

- 本游戏无终局胜利；分数、连击、角色收藏和距离排行榜构成持续目标。
- 完全错过下一站台时角色坠入云海，0.45 秒后进入结算卡。
- 结算卡展示本局分数、历史最佳、获得车票、再跳一次、角色收藏和排行榜入口。

## 6. Sound Effects

- 开始蓄力：低通三角波，约 180–420 Hz，随蓄力上扬，最大 1.05 秒，音量 0.05。
- 起跳：短促上行音色，约 440–880 Hz，80–160 ms，音量 0.10。
- 普通落点：柔和噪声与正弦音，约 440 Hz，100 ms，音量 0.08。
- 完美落点：660–1200 Hz 上行双音，140 ms，音量 0.14；配合金色粒子。
- 坠落：180–320 Hz 下行噪声，180–320 ms，音量 0.13。
- 解锁与装备：本次改动沿用无阻塞 UI 反馈；未来加入音效时应使用 740–1240 Hz、120 ms 的两段上行提示，并尊重静音设置。
