# Visual Direction

## 1. Thesis

“云端通勤，下一站更远。”Sky Leap 将蓄力跳跃包装成一段可收藏的末班车旅程：每个角色是一位轮廓鲜明的通勤者，车票来自每一次跑局，而非随机抽取。

## 2. Composition / Camera

正交等距镜头固定追随角色和下一座站台。游戏 HUD 只占顶部安全区；角色收藏入口左上、排行榜右上。商店是从底部升起的站台衣柜：当前角色占中心主舞台，前后角色在两侧局部露出，形成连续收藏册而不是卡片列表。

## 3. Color

- 云海：`#AED4F1`、`#CFE4F5`、`#FFD9E7`
- 墨色文字：`#3A2C52`；辅助文字：`#6C5F88`
- 操作和装备强调：`#FF4D97` / `#EC3A86`
- 车票：`#FFCF4D`。商店不再按角色切换整块界面色；固定使用云雾白、薰衣草 `#F5EFFF`、粉红 `#FF4D97` 与紫墨 `#5A4376`，让角色本身成为变化的颜色焦点。

## 4. Typography

标题与数字使用 Fredoka 700；中文回退 PingFang SC。HUD 数字 66 px，商店标题 27 px，角色名 17 px，说明文字不小于 13 px。

## 5. Shape, Material, Light

角色、站台与图标采用圆角方块和低多边形剪影。界面使用半透明云雾玻璃、1 px 白色高光边和柔和紫蓝投影；不使用金属写实、厚重霓虹或写实照片。

## 6. Assets

六位收藏角色使用现有 Three.js 方块模型：早班店长、赶课学生、夜班主厨、闪送骑手、消防英雄、末班幽灵。商店中央使用实时 Three.js 画布，在粉紫摄影棚灯光下缓慢转身并轻微浮动；两侧使用同模型静态预览露出前后角色。入口只用对应的单色 SVG 剪影作为快速识别标记，不使用 Emoji 或抽卡插画。

## 7. UI and Icons

所有功能性图标均为内联 SVG：角色剪影、车票和现有皇冠。收藏入口及所有操作按钮最小高度为 44 px；角色卡使用文字和颜色以外的“已装备/价格”状态，避免只依赖颜色。

## 8. Motion / VFX

商店 300 ms 上升进入、220 ms 淡出；中央角色以每秒约 0.48 弧度缓慢旋转并以 0.025 world-unit 上下浮动，角色切换从正面重启展示；减少动态偏好下停止循环旋转。蓄力、起跳、完美落点和坠落继续使用现有的挤压、轨迹、粒子和短暂慢动作。

## 9. Translated References

参考“天天过马路”的清晰方块角色剪影与收藏欲，但不复制其道路场景；参考“挤地铁”式通勤角色群像，将其转换为云端站台旅行主题。

## 10. Anti-patterns

禁止每局随机换角色、盲盒概率、功能 Emoji、通用深色赛博商店、与游戏模型不一致的写实人物头像、卡片的过度渐变与大面积发光。

## 11. Vertical-slice Acceptance

在 390×844 和 320×568 视口验证：待机时可看到当前角色和车票；打开商店后完整看到当前真实角色预览、两侧相邻角色、锁定价格和已装备状态；可切换至任一锁定角色并清楚比较造型；装备角色后回到画面立即更新；完成一局后车票增加且重开不随机换人；游戏、结算与商店不发生误触跳跃。
